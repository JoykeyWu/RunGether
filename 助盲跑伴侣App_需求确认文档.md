## 第一部分：已确认信息

### 项目概述

| 项目 | 确认内容 |
|------|----------|
| App 名称 | RunGether（助盲跑伴侣） |
| 项目主题 | 助盲跑伴侣 Android App（视障跑者与陪跑员协同跑步） |
| 项目性质 | 课程大作业 |
| 开发平台 | Android，最低支持 Android 7.0、目标 Android 14 |
| 必须满足 | 课程四个板块的全部要求：需求文档、从需求到设计、数据网络层、UI 集成与核心流程 |

### 用户角色与双端界面

| 项目 | 确认内容 |
|------|----------|
| 视障跑者端（盲人端） | 极简界面、超大按钮、高对比度配色、以语音操作为主 |
| 陪跑员端（引导端） | 方向摇杆、快捷指令按钮、跑步数据面板、地图 |
| 界面方案 | 两个身份、两套独立界面；首次启动选择角色，设置页可随时切换 |

### 核心功能范围（MVP）

作业板块二要求确定 3-5 个必做核心功能，本项目确定为以下 5 项：

| 序号 | 核心功能 |
|------|----------|
| 1 | 蓝牙配对连接 |
| 2 | 方向摇杆引导 |
| 3 | 语音播报 + 震动反馈 |
| 4 | 紧急求助 |
| 5 | 跑步数据记录与历史 |

### 技术方案

| 项目 | 确认内容 |
|------|----------|
| 开发语言 | Kotlin |
| 界面框架 | 传统 XML 布局 |
| 架构模式 | ViewModel + LiveData/StateFlow + Repository（作业要求） |
| 蓝牙通信 | 经典蓝牙（串口协议），传输 JSON 格式指令 |
| 本地数据库 | Room（作业要求） |
| 网络层 | Retrofit，含 2-3 个接口方法，用于跑步记录的上传/拉取；接口由在线 Mock 平台 mockapi.io（公网地址，国内可直连访问）提供。项目根地址：`https://6a0d90e6769682b8ee766f58.mockapi.io/api/v1/`，开发时写入 `local.properties` 或 `Constants.kt`，避免硬编码 |
| 语音播报 | Android 系统内置语音合成 |
| 震动反馈 | Android 系统内置震动 |
| 定位 | Android 原生 LocationManager（GPS Provider）获取定位，无需第三方 SDK 与 API Key；户外跑步场景下精度可满足轨迹绘制需求 |
| 轨迹图 | 自定义 Canvas 控件绘制跑步路线，不使用第三方地图底图，仅呈现轨迹形状与实时位置（无真实街道底图） |
| 方向摇杆 | 自定义控件（触摸事件 + 角度计算） |
| 摇一摇检测 | 系统加速度传感器 |
| 单元测试 | 针对数据库增删改查与 Repository 逻辑，≥3 个测试方法且全部通过（作业要求） |

### 盲人端核心功能

| 项目 | 确认内容 |
|------|----------|
| 主界面 | 两个超大按钮：开始跑步、紧急求助 |
| 操作方式 | 信息以语音播报（TextToSpeech）输出；盲人通过超大按钮 + 震动反馈操作，不含语音识别指令 |
| 开始 / 结束跑步 | 点击"开始跑步"后语音播报提示并开始计时与 GPS 定位；跑步中该按钮变为"结束跑步"，由盲人自己点击结束 |
| 指令接收 | 收到陪跑员指令后，语音播报对应内容并配合震动反馈 |
| 反馈规则 | 直行（不震动）、微调（短震1次）、转弯（短震2次）、大转弯（短震3次）、障碍（长震1次）、减速（短震2次）、停止（持续震动3秒） |

### 陪跑员端核心功能

| 项目 | 确认内容 |
|------|----------|
| 方向摇杆 | 圆形摇杆，按手指滑动方向与角度实时计算转弯指令；松手自动回中并发送"保持直行" |
| 快捷指令 | 障碍、减速、停止 三个按钮 |
| 数据面板 | 实时显示配速、累计距离、跑步时长，每秒刷新 |
| 实时轨迹 | 跑步过程中实时绘制移动轨迹，画面自动跟随当前位置；为自绘轨迹图，不含真实街道底图 |

### 蓝牙配对

| 项目 | 确认内容 |
|------|----------|
| 配对流程 | 陪跑员搜索附近设备并选择盲人端手机完成配对；盲人端语音提示连接状态 |
| 断线处理 | 自动重连；断线超过 10 秒语音提醒 |
| 状态显示 | 两端界面顶部实时显示连接状态 |

### 紧急求助

| 项目 | 确认内容 |
|------|----------|
| 触发方式 | 点击"紧急求助"按钮，或摇一摇（需明显用力、连续摇晃约 2 秒以上才触发，避免跑步晃动误触发） |
| 触发后动作 | 语音播报提示、手电筒闪烁、警报声、通知陪跑员端弹窗提醒 |
| 紧急联系人 | 保留；设置页可添加、编辑、删除紧急联系人（姓名 + 手机号） |
| 拨打紧急联系人电话 | 功能与界面均会做出来。但 Android 系统将"自动拨号"列为高敏感权限、安全策略严格拦截，实际不触发真实拨号，改以模拟方式演示（※ 拟做成仿真呼叫界面，如显示"正在呼叫 紧急联系人…"，此为开发方案，可调整）；项目提交时一并注明"受手机安全策略限制，自动拨打功能在真实环境较难实现" |
| 自动发送短信 | 不做 |

### 跑步记录与历史

| 项目 | 确认内容 |
|------|----------|
| 自动保存 | 每次跑步结束自动存入本地 Room 数据库 |
| 记录归属 | 盲人端与陪跑员端均保存跑步记录、均可查看历史；盲人端以语音方式浏览历史记录 |
| 记录同步（网络层） | 跑步记录通过网络接口上传与拉取，满足作业网络层要求；接口由 mockapi.io 在线 Mock 平台提供。Repository 作为单一数据源，先读本地数据库，再请求网络更新 |
| 记录字段 | 日期、总时长、总距离、平均配速 |
| 结算页 | 跑步结束后展示本次数据摘要 |
| 历史列表 | 按时间倒序，列表形式展示 |
| 历史详情 | 单次跑步详情 + 跑步轨迹 |
| 记录管理 | 长按记录弹出删除确认 |
